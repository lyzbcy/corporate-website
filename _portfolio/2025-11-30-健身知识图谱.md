## 1. 项目概览
本项目是一个基于 Web 的轻量化智能健身知识问答系统。它将专业的知识图谱技术应用于大众健身领域，通过直观的可视化图谱和智能对话界面，帮助用户快速查找训练动作、了解肌肉关联、制定训练计划。

项目突破了传统知识图谱系统依赖高配服务器和复杂数据库（如 Neo4j）的限制，创新性地构建了纯静态、无后端、零成本部署的解决方案，完全托管于 GitHub Pages，实现了“即点即用”的极速体验。

- **在线演示**: [点击访问 Fitness Knowledge Graph](#)
- **数据规模**: 包含 **800+** 训练动作，约 **6000** 个知识节点（涵盖肌肉、器材、难度等级），以及 **15000+** 条关联关系。

## 2. 核心技术点 (Technical Highlights)
本项目采用纯前端架构，实现了从数据存储到交互逻辑的完全静态化：

### 静态图谱数据存储 (Static JSON Graph)
- 摒弃了笨重的 Neo4j 图数据库，设计了扁平化的 **JSON 数据结构** (`graph_data.json`) 存储图谱数据。
- 通过 Python 脚本 (`build_graph.py`) 预处理海量异构数据，将其压缩为仅约 **300KB** 的静态文件，实现毫秒级加载。

### 客户端检索引擎 (Client-side Search Engine)
- 所有的搜索、过滤、关系推导逻辑均由浏览器端的 **JavaScript 引擎**在内存中执行。
- 实现了“零延迟”的交互响应，彻底消除了网络请求带来的查询等待。

### 交互式可视化引擎
- 基于 **ECharts** 实现高性能的力导向图 (Force-directed Graph) 渲染。
- 支持 **6000+** 节点的流畅展示，以及缩放、拖拽、点击高亮等复杂的交互操作。

### 中英双语映射机制
- 针对健身术语中英混用的特点，在前端构建了双向映射字典（如 `胸肌` <-> `chest`）。
- 实现了输入中文关键词即可索引底层英文数据的智能匹配功能。

## 3. 项目创新点 (Innovation Points)

### 3.1 架构创新：从“重后端”到“零后端”
传统的知识图谱应用通常需要部署昂贵的服务器（如 $20/月）和复杂的图数据库（Neo4j），维护成本高。本项目彻底去除了后端服务，将计算压力转移至客户端，实现了：
- **零成本部署**：依托 GitHub Pages 免费托管，无需购买任何服务器或数据库。
- **极简运维**：没有服务器宕机风险，只要 GitHub 不倒，服务就永久在线。

### 3.2 交互创新：双模式无缝切换
借鉴现代 APP 的设计理念，设计了两种互补的交互模式，满足不同用户需求：
- **图谱探索模式 (Graph Mode)**: 适合探索性学习。用户可以通过拖拽节点、查看连线，直观地理解“动作-肌肉-器材”之间的复杂关联。
- **全屏对话模式 (Chat Mode)**: 适合快速查询。仿微信的聊天界面，用户输入问题（如“练习胸肌”），系统即时反馈相关动作卡片，简洁高效。

### 3.3 领域迁移与数据重构
成功将医疗领域的知识图谱构建技术迁移至健身领域，证明了图谱技术在垂直领域的普适性。
- **数据模型重构**：将复杂的医疗“病-症-药”三角关系，重构为健身领域的“动作-肌肉-器材”多维关系。
- **结构化整合**：整合了 GitHub 开源社区的 `free-exercise-db` 数据，清洗并标准化了 800+ 个动作的说明、演示图和肌肉属性。

### 3.4 极致的用户体验
- **秒开体验**：得益于静态资源的 CDN 加速和数据压缩，页面加载速度远超传统动态网站。
- **隐私安全**：由于没有后端数据库，所有查询历史仅驻留在用户浏览器本地，完全不涉及用户隐私数据上传。

## 4. 总结
健身知识图谱项目是一个技术与实用性并重的创新尝试。它不仅为健身爱好者提供了一个免费、好用的查询工具，更在技术层面上探索出了知识图谱轻量化应用的新路径——即通过“静态数据+前端算力”替代“动态数据库+后端算力”，为个人开发者和小微应用的低成本落地提供了极具价值的参考范例。
