## 项目信息
- **类别**：学校作业
- **时间**：2024.12.21
- **角色**：核心开发者（代码实现、UI设计）

## 项目介绍

这是我的第一个完整的 Unity 游戏项目。虽然核心玩法是跟随教程实现的，但我在此基础上进行了大量的自主扩展和优化。

**主要功能扩展：**
- **完整游戏流程**：自主设计并添加了开始菜单、设置页面、游戏结束页面。
- **个性化功能**：增加了坦克换皮肤功能。
- **视听体验**：添加了完整的音效和背景音乐。

整体完成度高，在一众作品中脱颖而出，得到了老师的满绩认可。

## 🤕 遇到的困难

### 1. 团队协作与版本管理
项目初期是和好朋友两个人一起合作的。当时我们对 Git 等版本控制工具还不熟悉，导致代码合并非常困难。
*   **解决过程**：由于我的开发进度较快，最终决定代码开发部分主要由我完成，队友负责音效素材收集和实验报告撰写。这次经历让我深刻体会到了版本控制工具的重要性。

### 2. UI 自适应布局
在制作开始菜单和设置菜单时，完全是自己摸索的。最开始使用 Canvas 时，在我的电脑上显示正常，但换一台电脑后 UI 排版就完全乱了。
*   **解决过程**：通过查阅资料，我学习了 Unity UI 的自适应布局（Adaptive Layout）机制，并重构了所有 Canvas 的布局，确保在不同分辨率下都能正常显示。

### 3. 时间紧迫与团队精神
当时时间非常紧张，大部分小组只提交了简单的 Python 2D 游戏，但我们想做一点出类拔萃的东西。
*   **难忘经历**：我和好友连续三天在学院楼空教室熬夜开发到凌晨 4-5 点。虽然过程很累，甚至感觉"要死掉了"，但大家一起努力、半夜点夜宵的经历非常难忘。期间还有另一位好朋友特意来陪我们，帮忙测试游戏，这段共同奋斗的时光其实非常有趣。
