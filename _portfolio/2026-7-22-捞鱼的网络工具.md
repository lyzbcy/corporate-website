## 📊 项目信息
- **完成时间**: 2026年7月（持续更新中）
- **项目类型**: macOS 原生网络工具
- **核心技术**: Swift, WKWebView, macOS 原生命令（networksetup/scutil/route/dscacheutil）, setuid 特权助手
- **项目开源**: [GitHub 源码](https://github.com/lyzbcy/nettools) · [立即下载](https://github.com/lyzbcy/nettools/releases/download/v4.3/nettools-v4.3.1-macOS.zip)

## 💡 这个工具解决什么问题

网页打不开？DNS 出错？VPN 和梯子打架？公司内网访问不了？

这个工具用 macOS 原生命令实现了 Windows「360 断网急救箱」等效的能力：DNS 诊断修复、代理劫持清除、WiFi 体检、网速测试、公网连通守护，以及公司 VPN 与梯子的真正共存。

## 2. 核心功能

### 🚀 一键智能诊断
8 项只读体检：本机网络、默认路由、公网连通、DNS 解析、网页访问、系统代理、hosts 文件、防火墙。不盲目操作，先诊断再修复，找出网页打不开的真正原因。

### 🛡️ 公网连通守护
每 30 秒自动检查公网连通性。断网时按 **DNS → 代理 → DHCP** 顺序对症修复，每步后立即复测。安装免密授权后可真正无人值守——这是专门为「人不在电脑前保活远程连接」设计的。

### 🧹 修复工具箱
- 清空 DNS 缓存
- 换公共 DNS（阿里/腾讯/114/Google）
- 临时关闭失效代理（可一键恢复）
- 续租 DHCP

### 📶 WiFi 体检 + 测速
信号强度 RSSI、噪声、信噪比、信道、速率、安全协议一目了然。Cloudflare 测速源。

### 🔒 VPN 与梯子共存
用**精确路由 + 分域 DNS + 代理直连**三层分流，让公司内网和梯子各走各的路：
- 公司网段走 VPN，公共网络走本地
- 公司域名只问公司 DNS（macOS `/etc/resolver` 分域机制），绕过其他 DNS 的污染
- 公司域名加入代理直连名单，梯子不抢内网流量

### 🔐 免密授权
内置一个 setuid root 的特权助手（固定 action 白名单，拒绝任意命令）。一次安装永久免密，所有操作和守护都不再弹密码框。

## 3. 架构设计

采用 **Swift + WKWebView 壳 + setuid 特权助手** 的分层架构：

- **主程序**（Swift）：WKWebView 加载前端 `index.html`，通过 JS 桥接调度所有功能。普通用户权限运行。
- **前端**（单文件 HTML）：lyzbcy 视觉系统，6 个功能页面，守护状态机。
- **特权助手**（`netrepair-helper`）：独立的 setuid root 二进制。主程序调用它执行需要 root 的网络操作。固定 action 白名单 + 参数严格校验，安全且免密。

## 4. 🤕 技术难点

### DNS 污染与 VPN 共存
公司内网域名（如 `gitlab.corp.example.com`）被公共 DNS 解析成错误的公网 IP，导致 VPN 连着却访问不了内网。而 Tailscale 等工具的 MagicDNS 又会劫持所有 DNS 查询，公司 DNS 设成全局会导致公共网站超时。

**解决方案**：用 macOS 原生的 `/etc/resolver/<域名>` 分域 DNS 机制——只让公司后缀的查询走公司 DNS，其他查询保持原样。两边互不打架，谁都不用关。

### 守护的无人值守
守护的设计目标是「人不在电脑前保活远程连接」，但 macOS 每次特权操作都弹密码框，人不在谁输？

**解决方案**：setuid root 助手。一次安装后，守护触发的所有修复（清 DNS、关代理、续租 DHCP）都以 root 权限免密执行。
